Questions

Common questions about Claude Code — answered with direct tips and source links.

How do I prevent Claude from modifying files outside my task scope?

Be explicit in your prompt: "Only modify files in src/components/. Do not touch tests or config." Claude respects explicit scope boundaries.

What's the difference between /clear and starting a new session?

/clear removes conversation history but keeps the terminal session, loaded CLAUDE.md, and MCP connections. A new session reloads everything from scratch.

How do I use Claude Code in a CI pipeline?

Use the --print flag for non-interactive output. Set exit codes with --exit-code. See the automation tips section for patterns.

What is no-babysit mode?

A community convention (🚫👶) marking tips that require Claude Code experience to use safely. These tips may skip confirmation steps or use advanced flags.

How do I keep Claude from hallucinating file paths?

Always paste the actual file path from your editor. Ask Claude to verify a file exists with Read before editing it. Never describe a path verbally.

Can I use Claude Code offline?

No — Claude Code requires Anthropic API access. There is no offline mode as of 2025.
