workflow 🚫👶 no-babysit

Use /clear between unrelated tasks to prevent context contamination

When switching to a completely different task, clearing the conversation prevents Claude from applying patterns or assumptions from the previous context.

B Boris Cherny
context 🚫👶 no-babysit

CLAUDE.md is auto-loaded — use it for persistent project context

Put architecture notes, team conventions, and project-specific constraints in CLAUDE.md. Claude reads it automatically at session start without requiring you to paste it each time.

B Boris Cherny
security

Prefer allowlisting over blocklisting for tool permissions

Explicitly define which tools Claude can use rather than trying to enumerate dangerous ones. An allowlist approach is more predictable and secure.

T Thariq Shihipar
automation 🚫👶 no-babysit

Use --print flag for non-interactive CI pipelines

Run Claude Code headlessly: cc --print "your task". Suitable for scripted environments, CI/CD, and any context where interactive prompts aren't possible.

B Boris Cherny
context

Be explicit about what files are in scope for a task

When asking Claude to modify code, specify which files should and shouldn't be touched. Ambiguity leads to unexpected changes in unrelated files.

B Boris Cherny
git

Create a new branch before starting any significant task

Working on a dedicated branch makes it trivial to review changes, revert if needed, and keep the main branch clean. Ask Claude to do this as part of task setup.

L Lydia Hallie
mcp 🚫👶 no-babysit

Scope MCP servers to the minimal permissions required

When configuring MCP servers, grant only the specific resource access the task requires. A Notion MCP shouldn't have write access to unrelated databases.

B Boris Cherny
hooks 🚫👶 no-babysit

Use hooks for automated behaviors, not memory or instructions

Hooks in settings.json execute shell commands in response to events. They're the right place for "run linter after each edit" — not Claude's memory system.

B Boris Cherny
debugging

Paste actual error messages, not paraphrases

Give Claude the exact error output including stack traces. Paraphrasing an error strips the precise signal Claude needs to identify the root cause accurately.

C Cat Wu
agentic 🚫👶 no-babysit

Break long autonomous tasks into checkpoints with explicit verification steps

For complex multi-step tasks, structure the prompt to include verification gates: "complete X, then confirm Y before proceeding to Z." This prevents error cascades in long agentic runs.

D Dex Horthy
prompting

State the desired outcome, not just the action

Instead of "refactor this function", say "refactor this function so it's easier to unit test in isolation, without changing its public interface." The goal guides better decisions.

D Dani Avila
