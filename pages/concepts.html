Concepts

Core concepts you need to understand to use Claude Code effectively. These are the building blocks behind the tips.

Context Window

The context window is Claude's working memory — everything it can "see" at once. Managing what goes in determines what Claude knows, what it forgets, and how well it performs on long tasks. Tips like /clear and CLAUDE.md directly interact with this concept.

Agentic Loop

Claude Code operates in a plan-execute-observe loop: it plans actions, uses tools to execute them, observes results, and iterates. Understanding this loop helps you write prompts that work with — not against — Claude's reasoning process.

agentic tips →

Tool Use

Claude Code has access to built-in tools (Bash, Read, Edit, Grep, etc.) and can use MCP servers for extended capabilities. The permission model controls what tools are available and under what conditions. Understanding tool scope prevents both over-permission and capability gaps.

security tips →

MCP (Model Context Protocol)

MCP is the protocol that lets Claude connect to external systems — databases, APIs, file systems, SaaS tools. Each MCP server exposes a set of tools Claude can call. Scoping MCP permissions correctly is one of the most security-critical decisions in any Claude Code setup.

mcp tips →

Hooks

Hooks are shell commands configured in settings.json that execute automatically in response to Claude Code events (e.g., after a file edit, before a tool call). They're the mechanism for "automated behaviors" — not Claude's memory.

hooks tips →

CLAUDE.md

A special file that Claude Code loads automatically at session start. It's your persistent project context — architecture decisions, team conventions, constraints, and anything Claude should always know. Think of it as a CLAUDE.md as a commit message to your AI collaborator: write the why, not the what.

Headless / Non-interactive Mode

Claude Code can run without a terminal UI using the --print flag. Output goes to stdout, exit codes signal success/failure. This is the foundation of using Claude Code in CI/CD pipelines, scripts, and automated workflows.

automation tips →

Permission Model

Claude Code's permission system controls which tools require user approval and which are automatically allowed. The permission model operates at three levels: global (user settings), project (.claude/settings.json), and session (inline overrides). Understanding the hierarchy prevents both excessive prompting and unintended automation.
